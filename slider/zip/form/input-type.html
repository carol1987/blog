特殊元素

  • (1)form 中第一个 control, type=text 的 input 元素, 并且 name=isindex 时, 提交时, 仅提交 value 值.
  • type=hidden 的 input 元素; <input type="hidden" name="name" value="xxx" />
  • type=hidden 的 input 元素; <input type="hidden" name="_charset_" /> 不设置 value, 提交时取页面字符集.

text/search/url/email/password

type 属性值为 text/search 在约束上没有区别; 在界面显示上, search 会更像 search!(这个好难理解,ex: 输入框内有文本时, 框右侧会出现关闭[/清空]按钮)

input 元素. type=text 或者忽略 type 属性

因为电话号码格式在不同地区是不同的(风俗习惯/历史原因), 因此对 type=tel 格式不做约束验证(ps: 约束可使用 pattern)

email/url 和 tel 就不同了, 他们有统一的约定. 因此存在约束.

日期/时间相关

HTML 中支持 min, max, step 属性; JavaScrip api 中支持, valueAsDate, valueAsNumber 属性 和 stepUp(), stepDown() 方法.

特殊 UI 元素

让用户选择颜色, 并且存在默认值. 不支持时, 回退到 text 状态.

checkbox - 多选按钮

存在两个状态值(true/false [checked/unchecked]) 由属性 checked 指定; 三种状态(选中, 未选中, 半选中 - 此状态由 JavaScript 实现[elem.indeterminate = true]).

checkbox 和 radio 元素如果在 html 中不指定 value 属性时, 使用 JavaScript 取 value 时, 得到字符串 'on'. (不管当前元素处于什么状态)

radio 单选, 要求 name 值大写不区分

文件上传

添加 multiple 属性, 支持多文件上传(即在弹出选择窗口时, 可选择多个文件)

表单按钮 - 提交/重置/其他

只有存在 formowner 时, 才起作用.(formowner: 存在 form 属性指定或在 Tree 上存在 form 元素[标签])

type 属性值为 button 时, 默认无交互行为

Button 标签 - type: reset/submit/default

radio 在 form 外时, 测试情况

form 外的 radio(即, 不存在 form owner); radio 单选, 要求 name 值大写不区分

form 外的 radio, 但 input 指定相同的 form 属性值; radio 单选, 要求 name 值大写不区分

form 外的 radio, 但 input 指定不同的 form 属性值; radio 单选, 要求 name 值大写不区分
