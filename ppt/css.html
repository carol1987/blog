    • Block Formatting Contexts

    块级格式化上下文, 有自己独立的布局环境和规则, 它不受外部环境影响, 也不会影响外部环境; 可以包裹内部的 float 元素, 同时不与外部 float 元素重叠.

    触发条件

    1. float: 除 none 以外的值 - 浮动元素
    2. position: absolute, fixed - 绝对定位元素
    3. display: table-cell(table), table-caption, inline-block - 非块级盒子的块级容器
    4. overflow: 除了 visible 以外的值( hidden/auto/scroll )

    clearfix

    
.clearfix:before,
.clearfix:after {
  content: '';
  /* 创建 BFC 元素, 隔离 top/bottom 元素, 不让 margin 重叠 */
  display: table;
  /* 其他方法:
    display: block;
    height: 0;
    overflow: hidden;
  */
}
/* 清除浮动 */
.clearfix:after {
  clear: both;
}
/* IE 6/7 */
.clearfix {
  *zoom: 1;
}

    SASS

    说在开头:
    CSS 可以解决时, 不必要使用预处理工具.
    当在使用 CSS 的很复杂或文件很大, 不再容易维护时, 可使用工具来辅助开发.

    SASS: CSS 预处理器, 即在上线部署之前, 把 SASS 文件编译成 .css 文件.

    • SASS 的 .scss 格式兼容 CSS 语法.
    • chrome 通过 sourceMap, 支持调试 sass 文件.

    Install

    1. install:
      1. install ruby
      2. gem install sass
    2. command:
      1. sass --watch index.scss:output.css
      2. sass --watch app/sass:public/css
      3. --style nested/compact/expanded/compress

    Homework

    • 有一个容器, 左右两列组成
    • 左侧是图片, 右侧是内容(有可能很复杂)
    • 左右两侧存在间隙 - ex: 10px;
    • 左侧图片的尺寸不固定( width/height 都不固定 )
    • 右侧内容不能环绕左侧图片
    • 右侧要求占满余下空间
    • 右侧内容可能超出容器 - ex: New! 小图标
