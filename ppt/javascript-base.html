严格模式

  • 严格模式会把 JavaScript "陷阱"做为错误对待
  • 严格模式修正了一些引擎难以优化的错误, 同样的代码有些时候严格模式会比非严格模式下更快
  • 严格模式禁用了一些有可能在未来版本中定义的语法

开启严格模式方法:
在文件/函数最前面, 添加 "use strict";
不支持严格模式的浏览器会自动把字符串 "use strict" 忽略.
建议在严格模式下进行开发 - 养成良好编码习惯

数值转换 - Number( value ) & +value

  1. 为 Number 类型, 直接返回 value
  2. 为 Boolean 值, 则 true -> 1; false -> 0;
  3. 为 null, 返回 0
  4. 为 undefined, 返回 NaN
  5. 如果为字符串:
    1. 空字符串, 返回 0
    2. 如果字符串中只有数字(可包含前导符号), 则直接转换为数字, 如果存在前导零, 直接忽略.
    3. 如果字符串中包含浮点格式(2.3), 规则同上.
    4. 如果字符串中包含十六进制格式(0xf), 则将其转换为相同大小的十六进制整数值.
    5. 如果字符串包含其他格式字符, 返回 NaN
  6. 如果是 Object 类型, 则调用 .valueOf() 方法, 然后按照上面的规则转换. 如果转换的结果为 NaN, 就再使用 .toString() 方法, 再按照上面的规则转换.

数值转换 - parseInt( value, base )

把以 base 进制表示的值 value, 转换为 10 进制表示.

  1. 空字符串时, 返回 NaN
  2. 第一个字符不是数字或者正负号, 就返回 NaN.
  3. 第一个字符是数字字符, 就处理第二个, 直到不是数字为止. parseInt( '123a', 10 ); //123
  4. 它不认为小数点是合法字符

逻辑表达式

  • &&
    1. 第一个操作数为 false/0/''/NaN/null/undefined 时, 返回 false/0/''/NaN/null/undefined
    2. 其它, 返回第二个操作数(不一定是 Boolean)
    3. 发生短路操作, 即第一个操作符为 false 时, 不再计算第二个操作数(表达式)
  • || 同 && 类似
  • !
    1. 空字符串/数值 0/null/NaN/undefined, 返回 true
    2. 其他, 返回 false
    3. 应用 !!, 相当于把操作数转换为 Boolean

===, !==

=== 数据类型和值都相等; 其它为 !==

==, !=

  • 返回值为 Boolean 类型(true/false)
  • 如果有一个操作数为 Boolean, 则把 true/false 转化为 1/0
  • 字符串和数字比较, 则把字符串转化为数字
  • 如果一个操作数是对象, 另一个不是对象, 则调用对象的 .valueOf() 方法, 得到的值按上面规则再比较
  • null 和 undefined 相等
  • NaN 和任意值比较都不相等

>, >=, <, <=

  • 操作数有 NaN/undefined, 直接返回 false
  • 操作数都是数字, 直接比较
  • 操作数都是字符, 按字符编码比较
  • 操作数是数字和字符串时, 把字符串转化为数字, 再直接比较
  • 操作数为 boolean 时, 先把它转化为数字, 在按上面规则比较
  • 其它, 先 .toString, 后 .valueOf 转化, 再按上面规则比较
